# 프로젝트 목표

지속적인 AI를 통한 프로그램/앱/코드 생산으로 생산성이 매우 증가했지만, 계속해서 일회성으로 사용하고 잊혀집니다.
이 때문에 이전 작업물과의 일관성 확보를 위해 다시 코드를 찾아보거나 새롭게 코드를 작성하는 등, 시간 낭비가 지속적으로 발생합니다.
본 프로젝트는 이렇게 생산된 단일 html 웹 앱/파이썬 코드/CMD 커맨드를 보관하며 언제든 사용할 수 있는 아카이브를 만드는 것입니다.

## 주요 과제

1. 기능색인

단순한 아카이브로서만 사용할 것이었다면, 폴더를 만들어 던져넣기만 했을 것입니다.
코드/앱의 메타데이터(기능, 작동방식 등 포함)를 입력하고 저장할 수 있어야 하고, 이 메타데이터를 기반으로 검색 가능해야 합니다.

2. 즉시 실행

아카이빙된 코드는 즉시 실행 가능해야 합니다.
이 기능은 python 등의 단순 실행 뿐만 아니라, 하나의 html로 작성된 웹앱과 같은 경우에도 즉시 실행하여 기능을 사용할 수 있어야 합니다.
웹앱은 여러 창으로 열 수 있어야 합니다.

3. 매개변수
이 프로젝트에서 다룰 코드 대부분은 하드코딩 된 설정, 인풋을 사용하기 마련입니다. 따라서 사용자가 어떤 설정을 하려면 내부에 하드코딩된 정보를 수정해야 할 수 있습니다. 사용자는 코드 내에 존재하는 하드코딩된 설정에 대하여 매개변수를 지정하고, 실행 전에 수정할 수 있어야 합니다.

- 예시

``` 
setting 1 = "blabla"

if __name__=="__main__":

input 1 = "file.something"

run(input 1)

```
매개변수 설정 후
``` 
setting 1 = {setting 1}

if __name__=="__main__":

input 1 = {input 1}

run(input 1)

```
매개변수는 코드 바깥에서 수정하여 작성가능
setting 1 = blabla
input 1 = file.something

예시는 파이썬을 기준으로 작성되었으나 이런식의 작동이 범용적으로 가능해야 합니다.
